# Plan: Fix Theme Not Showing Offline

## Problem Statement
The theme does not show properly when the app is offline. Specifically, when a custom background image theme is selected (especially network-based), and the app goes offline without the image cached, the theme falls back to a default gradient instead of preserving the user's selected gradient.

## Root Cause Analysis
- ThemeProvider stores theme settings in local SharedPreferences.
- For custom themes with network images, ThemeCacheService caches images locally.
- If a network image is not cached and the app is offline, backgroundDecoration falls back to the default gradient [AppColors.surfaceDark, AppColors.primaryDark].
- However, when setBackgroundImage is called, _gradientColors is set to null, so even if the user had a custom gradient before selecting the image, it's lost.
- This results in the theme not showing the user's intended gradient when offline.

## Minimum Code Changes
- Remove the line `_gradientColors = null;` in ThemeProvider.setBackgroundImage() to preserve any existing gradient as fallback.
- This ensures that if a network image fails to load (e.g., offline), the user's previously selected gradient is used instead of the default.

## Implementation
Edit lib/providers/theme_provider.dart:
- In setBackgroundImage method, remove line 195: `_gradientColors = null;`

## Verification
- Test with a network theme selected and cached → should work online.
- Go offline without cache → should fallback to user's gradient if previously set, else default.
- Ensure setting a new gradient still clears image, and setting image preserves gradient for fallback.
